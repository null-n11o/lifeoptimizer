# 執筆ルール - ブログ記事スタイルガイド

## 📝 文体の特徴

### 1. 口語的でフランクな語り口

**基本方針**
- 読者に直接語りかけるような親しみやすい文体
- 堅苦しすぎず、かといってカジュアルすぎない絶妙なバランス
- 敬語と口語のミックス

**使用する表現**
- 「〜ですね」「〜と思います」「〜でしょう」
- 「めちゃくちゃ」「かなり」「ガチで」「まさに」「スーパーやばいです」
- 「〜してみてください」「〜しましょう」「ぜひ参考にしてください」

**具体例**
✅ 良い例:
- 「個人的にはこの機能が一番使いやすいと思います」
- 「これはかなり重要なポイントですね」
- 「SEOをガン無視して、ガチの個人的に厳選した内容をシェア」
- 「まあこれは運用する中で腹落ちしてくることなのですが」

❌ 避ける例:
- 「〜ということをご承知おきいただきたいです」（堅すぎる）
- 「マジでヤバい」（カジュアルすぎる）

### 2. 主観と客観の明確な使い分け

**基本方針**
- 客観的事実と主観的意見を明確に区別する
- 個人的な意見には必ず「個人的に」「自分は」などの前置きをつける

**使用する表現**
- 「個人的に〜」「自分は〜」「私も〜」
- 「〜かもしれません」「〜でしょう」（断定を避ける場合）
- 「〜と言えるでしょう」「〜だと思います」

**具体例**
✅ 良い例:
- 「個人的にはObsidianが一番使いやすいと感じています」
- 「セネカによれば、〜と語っています（客観）。これは現代にも通じる考え方ですね（主観）」
- 「2025年は「AIエージェント元年」と呼ばれています（客観）。個人的にもこのトレンドに乗っておくべきだと思います（主観）」

### 3. リズム感のある文章構成

**基本方針**
- 短文と長文を織り交ぜてリズムを作る
- 一文が長くなりすぎないように注意
- 箇条書きを積極的に活用

**パターン**
```
短文で結論。長文で説明を加え、その理由や背景を述べる。また短文で締める。
```

**具体例**
✅ 良い例:
- 「Obsidianは最強のノートアプリです。ローカルで動作するため高速で、プラグインによる拡張性も高く、完全無料で使えます。個人的に3年使っていますが、手放せません。」

## 🏗️ 記事構成のテンプレート（基本は各カテゴリーのファイルを参照）

- AI・デジタルツール活用に関する記事: templates/tools.md
- 読書まとめ記事: templates/books.md
- 企業分析記事: templates/companies.md
- ビジネス・テクノロジー用語解説記事: templates/concepts.md
- カルチャー(個人的に好きな映画・ドラマ、スポーツ、マンガの解説記事): templates/culture.md

### 基本構造

```markdown

## 導入部分（問題提起・共感）

## 【商品名/サービス名/概念名/企業名/人物名/書籍名】の概要

### 特徴1

### 特徴2

## 【メイントピック1】

### サブトピック1

### サブトピック2

## 【メイントピック2】

## 【メイントピック3】

## 個人的な使い方・感想

## まとめ: 要点を箇条書きで

```

### 1. 導入部分（冒頭）の書き方

**目的**
- 読者の共感を得る
- 記事の重要性を示す
- トレンドや話題性を提示

**パターン**
1. **問題提起型**
   - 「〜で悩んでいませんか？」
   - 「〜という経験はありませんか？」

2. **トレンド提示型**
   - 「2025年は〜の年と呼ばれています」
   - 「最近〜が話題になっています」

3. **個人体験型**
   - 「〜を使ってみたら、〜でした」
   - 「久しぶりに〜するほどハマりました」

**具体例**
```markdown
情報収集や検索の方法も生成AIによって大きく変わりつつあり、盤石かと思われた
Googleのビジネスモデルの転換も考えられています。

現在そのGoogleの一番の脅威となっているのがPerplexity AIです。従来の検索
エンジンとは一線を画す、この革新的なツールは、情報収集からコンテンツ作成
まで、幅広いニーズに応える可能性を秘めています。

今回はそのPerplexity AIの使い方をを徹底的に解説していきます。
```

## 💬 表現・言い回しの特徴

### 1. タイトルの付け方

**パターン**
```
【カテゴリ】タイトル本文 | サブタイトル
```

**使用する修飾語**
- 「最強の〜」「完全〜」「徹底〜」
- 「〜入門」「〜大全」「〜完全解説」
- 「爆速で〜」「〜を使いこなし」

**具体例**
- 【Obsidian】使い方大全 | 最強ノートアプリの使用方法を徹底解説
- 【Perplexity AI】の使い方を徹底解説 | 最強の検索AIを使いこなし爆速でコンテンツ作成
- Claudeの新機能「Projects」完全解説 - 最強AIはさらに一段階上のレベルへ

### 2. 強調表現

**よく使う強調語**
- 最強、最高、最適、最大
- 完全、徹底、詳細
- 革命的、画期的
- 圧倒的、爆速
- とんでもなく、めちゃくちゃ

**注意点**
- 乱用しない（1段落に1つまで）
- 根拠や理由と組み合わせる

**具体例**
✅ 良い例:
- 「現状最強レベルのモデルである、Claude 3.5 sonnetに加えて、Artifactによるコンテンツの生成・改善スピードがとんでもなく早くなる」（理由付き）

### 3. 引用の使い方

**パターン1: ブロック引用**
```markdown
> 引用文をそのまま記載
> 複数行にわたる場合もある
>
> 引用元: 〜〜〜
```

**パターン2: インライン引用**
```markdown
〜〜と語っています。セネカの言葉を借りれば「〜〜」ということです。
```

### 4. 読者への語りかけ

**基本パターン**
- 「〜してみてください」
- 「〜すると良いでしょう」
- 「ぜひ参考にしてください」
- 「〜してみましょう」

**問いかけ形式**
- 「〜ではないでしょうか？」
- 「〜だと思いませんか？」

### 5. 具体例の示し方

**パターン**
```markdown
例えば、〜〜のようなケース。

以下具体例。
- 例1
- 例2
- 例3
```

または

```markdown
具体的には以下のような形。

1. ケース1の説明
2. ケース2の説明
```

### 6. 数字の効果的な使い方

**基本方針**
- 具体的な数字で信頼性を高める
- インパクトのある数値で読者の注意を引く
- 期間や実績を明示する

**使用パターン**
```markdown
- 期間: 「10年以上」「3年間」「40歳手前」
- 金額: 「約10億円」「月額$20」「年間180ドル」
- サイズ/スペック: 「2m超え」「82試合」「30チーム」
- 実績: 「70冊の本を出版」「500の研究論文」
```

**具体例**
✅ 良い例:
- 「10年以上NBAを観続けている」（信頼性）
- 「平均年俸が約10億円という破格の待遇」（インパクト）
- 「レブロン・ジェームズなんて40歳手前なのに」（具体性）

### 7. 疑問形で共感を生む

**基本方針**
- 読者の心の声を代弁する
- 驚きや疑問を疑問文で表現
- 自問自答の形式で理解を深める

**使用パターン**
```markdown
- 「これ、〜なの？」
- 「〜ではないでしょうか？」
- 「〜だと思いませんか？」
```

**具体例**
✅ 良い例:
- 「これ、人間がやってるスポーツなの？」（驚き）
- 「まだ観たことがない人はマジでもったいない」→「これを観ないなんて、もったいないと思いませんか？」
- 「なぜこんなにハマるのか？それは〜」（自問自答）

### 8. 対比で理解を深める

**基本方針**
- before/afterで変化を示す
- 2つの視点を対比させる
- 「〜と、〜では全然違います」パターン

**使用パターン**
```markdown
- YouTubeのハイライトで観るのと、試合を通して観るのでは全然違います
- 日本のBリーグと、NBAでは全くレベルが異なります
- 〜と思っていたが、実際は〜だった
```

**具体例**
✅ 良い例:
- 「YouTubeのハイライトで観るのと、試合を通して観るのでは全然違います。ハイライトだと「すごいプレーの連続」に見えますが、実際の試合では「普通のプレーのレベルが異常に高い」ということに気づくはず」
- 「〜と思われがちですが、実際には〜」

### 9. 「〜わけです」で納得感を出す

**基本方針**
- 説明を締めくくる際に使用
- 因果関係を明確にする
- 納得感・理解を促進する

**使用パターン**
```markdown
- しかもそれが試合を通して続くわけです
- つまり〜ということわけですね
- だからこそ〜なわけです
```

**具体例**
✅ 良い例:
- 「ステフィン・カリーの3ポイントシュート。ハーフコートからでも平気で決めるし、ディフェンスが目の前にいても関係なくシュートを打って決める。しかもそれが試合を通して続くわけです」

### 10. 固有名詞で具体性を出す

**基本方針**
- 具体的な人名・商品名・サービス名を使う
- 読者が知っている/親しみのある例を選ぶ
- 専門性と親しみやすさのバランス

**使用パターン**
```markdown
- 有名人の名前: 「レブロン・ジェームズ」「サム・アルトマン」
- ツール名: 「Obsidian」「Notion」「ChatGPT」
- 企業名: 「Google」「Microsoft」「OpenAI」
```

**具体例**
✅ 良い例:
- 「サム・アルトマンやAnthropicのDarioは~」（具体的な人名）
- 「レブロン・ジェームズなんて40歳手前なのに、いまだに若手をぶち抜いてダンクを決める」（有名選手で例示）

❌ 避ける例:
- 「ある有名な選手は〜」（曖昧）
- 「あるツールでは〜」（具体性なし）

## 🚫 避けるべき表現・パターン

### 1. 堅苦しすぎる表現

❌ 避ける:
- 「本機能は〜であると考えられる」
- 「〜に関しましては」
- 「〜につきまして」
- 「誠に〜」

✅ 代わりに:
- 「この機能は〜です」
- 「〜については」
- 「〜について」
- （不要）

### 2. 曖昧な表現

❌ 避ける:
- 「一般的に〜と言われています」（出典なし）
- 「多くの専門家が〜」（具体的な名前なし）
- 「ある研究によれば〜」（研究の特定なし）

✅ 代わりに:
- 出典を明記する
- 具体的な人名・組織名を挙げる
- 参考文献をリンクする

### 3. ネガティブすぎる表現

❌ 避ける:
- 他のツール/サービスの批判
- 「〜は使えない」「〜は最悪」
- 「絶対に〜すべきでない」

✅ 代わりに:
- 客観的な比較
- 「個人的には〜の方が合いました」
- 「〜には向いていないかもしれません」

### 4. 過度なカジュアル表現

❌ 避ける:
- 「ヤバい」「普通に」（単独使用）
- 「〜っす」「〜じゃん」
- 顔文字、絵文字の多用（タイトルや見出し以外）

✅ 使える範囲:
- 「ガチで」「めちゃくちゃ」「マジで」は個人的感想や体験談で限定的に使用可
  - 例: 「マジでもったいない」「ガチでハマった」「めちゃくちゃ良い」
- 「半端ない」「本気で」などの感情表現も効果的
- 絵文字は見出しの先頭のみ（例: 📝、🏗️、💬）

## ✍️ 文章の細かいテクニック

### 1. 段落の作り方

**基本ルール**
- 1段落 = 1トピック
- 3-5文で1段落
- 段落間は必ず空行を入れる

**パターン**
```
【主張文】結論や要点を述べる。
【説明文】理由や背景を説明する。
【補足文】具体例や追加情報を加える。
```

### 2. 箇条書きの使い方

**使用タイミング**
- 3つ以上の項目を列挙する時
- 手順を示す時
- 比較項目を並べる時

**番号付きリスト（順序がある）**
```markdown
1. 最初のステップ
2. 次のステップ
3. 最後のステップ
```

**番号なしリスト（順序がない）**
```markdown
- 項目1
- 項目2
- 項目3
```

### 3. 強調の使い方

**太字（**〜**）**
- 重要な概念
- キーワード
- 結論部分

**イタリック（未使用）**
- 基本的に使わない

**マーカー的な使用**
- 箇条書きの先頭で太字を使う

```markdown
- **重要なポイント:** 説明文がここに入る
- **別のポイント:** 別の説明
```

### 4. リンクの貼り方

**内部リンク（自分の記事）**
```markdown
詳しくは以下記事で解説しています。

![](画像URL)

- [記事タイトル](記事URL)
```

**外部リンク**
```markdown
参考: [リンクテキスト](URL)

[タイトル | 出典元](URL)
```

### 5. 画像の使い方

**配置パターン**
```markdown
![説明文](画像URL)

説明文 - 出典元
```

または

```markdown
![](画像URL)

引用: 出典元
```

### 2. 参考文献（該当する場合）

```markdown
## 参考文献

- [タイトル](URL)
- [タイトル](URL)
- [書籍名](AmazonアフィリエイトURL)
```

## 📝 実践例

以下、実際の文章例を「改善前」「改善後」で示します。

### 例1: ツール紹介の冒頭

❌ 改善前:
```
本稿では、近年注目を集めているノートアプリケーション「Obsidian」に
ついて、その機能及び使用方法を詳細に解説する。
```

✅ 改善後:
```
デジタルな世の中になり、情報が溢れてくるにつれ、最近PKM（Personal
Knowledge Management）の方法論や様々なそれらを体現するツールが開発
されています。
その中でも最も有名なナレッジ管理手法として知られるのがZettelkasten。
そのZettelkastenを現代的に実現する上で最も理想的なツールが今回紹介
する「Obsidian」だと個人的に思っています。
```

### 例2: 比較の表現

❌ 改善前:
```
NotionよりもObsidianの方が優れている。Notionは動作が重く、使いにくい。
```

✅ 改善後:
```
NotionとObsidianは用途が異なります。
Notionは共同作業やデータベース機能が強力で、チームでの情報共有に
適していますが、動作がやや重く感じることがあります。
一方、Obsidianはローカルで動作するため高速で、個人の知的生産に
特化しています。個人的には、両方を用途に応じて使い分けるのが
最適解だと思います。
```

### 例3: まとめの書き方

❌ 改善前:
```
以上、Obsidianについて解説した。活用されたい。
```

✅ 改善後:
```
## まとめ: 自分に合ったObsidianの使い方を見つけよう
基本的にObsidianを使い始める前に覚えておくべきことは
1. フォルダ分けしない
2. 新しいノートには必ず何かをリンクさせる
3. トップダウン方式でノートをアトミック化していく
の3つだけで、その他の部分は私が個人的に運用している方法になります。
使う人によって合った方法というのが必ずあるので、色々な人のマネを
してみて、修正する。ということを繰り返して、自分に合ったObsidianの
使い方を見つけてください！
```
