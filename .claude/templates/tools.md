## 2. 概要セクションの書き方

**含めるべき要素**
- サービス/ツール/概念の定義
- 開発元・提唱者
- 主な特徴（3-5個）
- 基本情報（価格、対応プラットフォームなど）

**具体例**
```markdown
## Obsidianとは何か

Obsidianとは、個人の知識管理（PKM）ツールとして利用されているローカルで
動くマークダウンエディタです。

### 運営方針とビジョン
Dynalistを開発したShida LiとErica Xuという二人によって開発され...

### 主な特徴
- マークダウンエディター
- ローカルで動く
- ノート間のリンク機能
- プラグインによる高いカスタマイズ性
```

### 3. 比較表の活用

**いつ使うか**
- 複数のツール/サービスを比較する時
- メリット/デメリットを整理する時
- 機能の有無を明確にする時

**テーブル形式**
```markdown
| 機能 | ツールA | ツールB | ツールC |
| --- | --- | --- | --- |
| 価格 | 無料 | 月額$20 | 無料 |
| オフライン | はい | いいえ | はい |
| カスタマイズ性 | 高い | 中程度 | 高い |
```

### 4. 個人的な使用例・感想セクション

**目的**
- 信頼性の向上
- 具体的なユースケースの提示
- 読者との共感形成

**書き方**
- 「個人的に〜」で始める
- 具体的な数字や期間を入れる
- 実際のスクリーンショットを含める

**具体例**
```markdown
## 個人的なObsidianの使い方

これ以降はあくまで私の個人的なObsidianの使い方になります。

絶対に守るべきことは一つもないので、参考にしたいと思うところだけ参考にして、
あとはご自身に合う方法でObsidianの使い方をアレンジしていってください。

### 第二の脳: ノートを繋ぎ合わせる
一番オーソドックスな使い方。グラフビューで見るとまるで自分の脳内を見ている
かのようにノートの繋がりが可視化されます。
```

## 5. まとめセクション

**基本構造**
```markdown
## まとめ: 〜〜〜（要約を一文で）

- 要点1
- 要点2
- 要点3
- 要点4
- 要点5

（締めの一文）
```

**締めの一文のパターン**
- 行動喚起: 「ぜひ試してみてください」
- 展望: 「今後〜になっていくでしょう」
- 個人的感想: 「個人的に〜だと思います」
